Format - JSON

Import multiple items by following JSON conventions, i.e., enclosing comma-separated JSON objects in square brackets (i.e., [{item 1},{item 2},{item 3},...]).

A sample JSON item is shown below. The acceptable tags are: 'item_name', 'count', 'model_number', 'description', 'tags', 'custom_fields'.

NOTE: 'item_name' and 'count' are required, and 'count' must be a positive integer. If a tag which does not exist is input, it will be created in the system. However, if a custom field which does not exist is input, an error will occur.

{

'item_name': "str",

'count': int,

'model_number': "str",

'description': "str",

'tags': [

{
'tag':"str"
},

{

'tag':"str"
},...

],

'custom_fields': [

{
'field_name':"str",

'field_value':int/float/"str"

},...
]

}

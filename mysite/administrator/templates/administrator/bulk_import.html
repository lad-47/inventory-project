Format - JSON

Import multiple items by following JSON conventions, i.e., enclosing comma-separated JSON objects in square brackets (i.e., [{item 1},{item 2},{item 3},...]).

Items

A sample JSON item is shown below. The acceptable tags are: 'item_name', 'count', 'model_number', 'description', 'is_asset', 'tags', 'custom_fields'.

NOTE: When creating a normal Item instance, 'item_name' and 'count' are required, and 'count' must be a positive integer. If a tag which does not exist is input, it will be created in the system. However, if a custom field which does not exist is input, an error will occur.

NOTE: When creating an item instance for an Asset, 'count' should be omitted and 'is_asset' should be set to true, otherwise, it will be automatically set to false. In this case, Custom Fields provided should not be 'per_asset'.

{

'item_name': "str",

'count': int,

'model_number': "str",

'description': "str",

'is_asset': boolean (default = false)

'tags': [

{
'tag': "str"
},...

],

'custom_fields': [

{
'field_name': "str",

'field_value': int/float/"str"

},...
]

}

Assets

A sample JSON asset is shown below. The acceptable tags are: 'asset_name' and 'custom_fields'.

NOTE: An Item instance must exist with the name 'asset_name' before an Asset instance can be created.

NOTE: Custom Fields provided must be 'per_asset' fields.

{

'asset_name': "str",

'custom_fields': [

{

'field_name': "str",

'field_value': int/float/"str"

},...
]

}
